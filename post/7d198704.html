我的Hexo博客部署过程

发表于 2023-09-04 分类于 框架

什么是GitHub Pages？

GitHub Pages 是由 GitHub 官方提供的一种免费的静态站点托管服务，让我们可以在 GitHub 仓库里托管和发布自己的静态网站页面。

什么是Hexo？

Hexo 是一个快速、简洁且高效的静态博客框架，它基于 Node.js 运行，可以将我们撰写的Markdown 文档解析渲染成静态的 HTML 网页。

Hexo+GitHub Pages静态网站的文章发布原理

在本地撰写 Markdown 格式文章后，通过 Hexo 解析文档，渲染生成具有主题样式的 HTML 静态网页，再推送到 GitHub 上完成文章的发布。也就是hexo g生成静态网页，然后hexo d部署到GitHub上。

Hexo+GitHub部署博客的优点和缺点

优点：完全免费；静态站点，轻量快速；可按需求自由定制改造；托管在 GitHub，安全省心；迁移方便……

不足：发文不便，依赖于本地环境；更适合个人博客使用；GitHub 在国内访问速度时开始慢。

GitHub Pages与个人域名绑定

上述过程完成后就可以通过https://账号名称.github.io来访问博客了。如果我们有域名的话可以将域名绑定到这个地址。例如我是在阿里云买的一个域名，那么就可以在阿里云控制台中设置DNS解析。

可以设置类型为CNAME或者ipv4，如果使用CNAME就只需要指向账号名称.github.io，使用ipv4则是指向GitHub的ip地址，可以在cmd执行：

就可以查看到IP地址。

以上完成后还需要在GitHub配置仓库的domain，不然访问我们的域名还是会报404。

配置domain步骤：进入博客仓库–>Settings–>Pages–>Custom domain，填入我们的域名保存即可。注意这里的Settings跟账号的Settings不是同一个，而是仓库中的Settings。

至此域名绑定也完成了。

注意事项

我在使用butterfly后，文档建议新建一个配置文件_config.butterfly.yml，然后复制 _config.yml里面的内容过去，然后butterfly的一些配置是必须要写在 _config.butterfly.yml中的，否则会失效。例如index_img，也就是首页图片的配置，我写在 _config.yml是无法生效的。所以在参考butterfly文档配置时，尽量将配置内容写在 _config.butterfly.yml中。

博客名称的修改：不要直接去修改md文件的名称，是没用的，博客的名称、标签和发布时间的信息是在md文件里面的一开始中修改才能生效。

删除博客：直接删除md和资源文件。
